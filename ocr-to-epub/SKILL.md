# OCR to EPUB — 从 OCR 文本恢复电子书

将一本书的 OCR 扫描文本和目录信息，恢复为结构完整的 EPUB 电子书。

## 输入

两种信息缺一不可：

- **书籍正文**：OCR 扫描结果，通常是一个大文本文件（如 Markdown 或纯文本），内容按章节连续排列
- **目录信息**：章节标题和层级结构的列表。形式可以是：
  - 一个目录骨架文件（Markdown 列表）
  - 书籍的原始目录页 OCR 文本
  - 用户直接口述的章节结构

## 工作流程

### 1. 理解结构

先读取目录信息，厘清全书结构：
- 前置内容（鸣谢、前言等）
- 各部分/卷的划分
- 各章节标题及小节层级

同时快速扫描 OCR 正文，确认内容完整性，了解 OCR 噪声程度（字符识别错误、断行、乱码等情况）。

### 2. 提取章节

创建 `chapters/` 目录。按照目录结构，从 OCR 正文中找出每章的起止位置，逐章提取完整正文写入独立文件。

命名规范：`{序号}-{章节标题}.md`（如 `01-跨文化聚点研究.md`）。

如果前置内容（鸣谢、前言）和某章正文扫描在同一块连续文本里，可以在同一个源文件中按行号范围切分出不同的小节文件。

提取同时清理 OCR 噪声：
- 合并不当断行（通常是句中被切断的行）
- 修复断裂的英文/拼音单词
- 清除乱码字符和 OCR 标签残余
- **保留全部实质性论证内容**，不增删观点

### 3. 生成 EPUB

#### 方式一：使用 build_epub.py（推荐）

编写一个 `book.json` 配置文件，然后运行：

```bash
python3 build_epub.py book.json [chapters_dir] [output.epub]
```

`book.json` 的结构：

```json
{
  "meta": {
    "title": "书名",
    "creator": "作者",
    "publisher": "出版社",
    "date": "2023-09",
    "language": "zh-CN"
  },
  "chapters": [
    {"id": "ch00a", "title": "鸣谢",    "file": "00-鸣谢与前言.md", "lines": [1, 7]},
    {"id": "ch00b", "title": "前言",    "file": "00-鸣谢与前言.md", "lines": [10, null]},
    {"id": "ch01",  "title": "第一章 …", "file": "01-第一章.md"}
  ]
}
```

- `id`：EPUB 内部标识符，不可重复
- `title`：章节标题
- `file`：chapters 目录下的源文件名
- `lines`（可选）：`[起始行, 结束行]`，行号从 1 开始，`null` 表示到文件末尾。省略则读整个文件

#### 方式二：用其他工具

也可以使用 pandoc 等工具生成 EPUB，但需注意：
- 慎用 `---` 分隔线（pandoc 会将其解释为 YAML 元数据块边界，导致章节合并或丢失）
- 确保每个章节由一个 H1 标题开头，以便正确分章

### 4. 验证

生成 EPUB 后用 `unzip -l` 检查内部文件结构，确认：
- 章节数与目录一致
- 每个章节一个独立 xhtml 文件
- TOC（`toc.ncx` 或 `nav.xhtml`）完整列出所有章节标题
- 在阅读器（Apple Books 等）中打开确认可正常阅读

## 注意事项

- `build_epub.py` 零外部依赖，仅使用 Python 标准库（`zipfile`, `re`, `html`, `json`, `uuid`）
- Markdown → HTML 转换目前覆盖标题、段落、粗体、斜体、分隔线。不支持列表、表格、图片等复杂格式
- 如果 OCR 正文很大（>200KB），需要分批读取以保持在上下文窗口内
- 本技能的核心在于"识别结构 + 逐章提取 + 清理噪声"的流程，`build_epub.py` 只负责最后一步打包
